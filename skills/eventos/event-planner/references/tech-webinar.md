# Tech Checklist — Webinar Mode (online live)

Load this when `mode = webinar`. Run **at least 30 minutes before** go-live.

## Audio

```
[ ] Microphone connected and selected in platform settings
[ ] Test recording — play back 10 seconds to confirm clarity
[ ] Notification sounds silenced on all apps and phone
[ ] Input levels set (peaks around -12 dB, no clipping)
[ ] Headphones in (prevents echo from monitor speakers)
```

## Camera

```
[ ] Positioned at eye level; lighting on face, not behind
[ ] Background clean or simple virtual background (no flicker)
[ ] Preview checked in platform settings
[ ] Auto-focus locked (or manual focus set) to avoid mid-talk refocusing
```

## Screen Share

```
[ ] Slide deck on Slide 1, demo tabs open separately
[ ] All unrelated tabs, apps, and notifications closed
[ ] Do Not Disturb enabled on OS
[ ] Test share — confirm attendees see slides, not desktop
[ ] Presenter view / speaker notes accessible on a second screen
```

## Recording

```
[ ] Cloud or local recording enabled and tested
[ ] Backup recording via OBS / QuickTime / Loom
[ ] Storage space confirmed (recordings are large)
```

## Platform Settings

```
[ ] Attendees muted on entry
[ ] Chat enabled for all participants
[ ] Waiting room disabled (or staffed) for direct join
[ ] Co-host / moderator assigned
[ ] Reactions / polls / hand-raise enabled as the structure uses them
[ ] Q&A panel configured if used
```

## Engagement Tools

```
[ ] Polls pre-built and saved (one per engagement prompt)
[ ] Chat prompts scripted in speaker notes ("Type YES if...")
[ ] Breakout rooms pre-configured (if workshop-style segments)
```

## Backup Plan

```
[ ] Backup event link ready to email if platform crashes
[ ] Mobile hotspot as secondary internet connection
[ ] PDF version of slides if screen share fails
[ ] Phone dial-in as backup audio
[ ] Co-host briefed to take over if host drops
[ ] Pre-written "we're experiencing a tech issue, stand by" message
```

## Post-Show

```
[ ] Stop recording
[ ] Export chat log (attendee questions, URLs shared)
[ ] Note attendance peak vs. drop-off for follow-up targeting
[ ] Send replay the next day (NOT immediately — trains attendees to skip live)
```
