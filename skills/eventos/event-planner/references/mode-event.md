# Mode: Event (in-person / virtual / hybrid)

## Sub-type strategy shapes

| Sub-type | Duration | Shape |
|---|---|---|
| Conference | Full day+ | Keynote → parallel sessions → breaks → networking → closing |
| Meetup | 1–2 h | Welcome → featured talk → structured networking → open |
| Launch party | 2–3 h | Mingle → product moment → demo → celebration |
| Networking | 1.5–3 h | Icebreaker → round 1 → break → round 2 → closing |
| Summit | Multi-day | Themed tracks per day → keynote + sessions + social |

Produce a **high-level session schedule** (sessions + owners, not minute-by-minute — that belongs in `event-run-of-show`).

## Master Checklist (event spine)

**4+ weeks before:**
- Confirm venue, insurance, permits
- Confirm speakers, sign releases, collect bios + headshots
- Draft agenda; set up registration page (delegate copy)
- Create budget tracker (see `templates.md`)
- Order branded materials, signage, name tags
- Book catering / AV vendor / photographer

**2–3 weeks before:**
- Launch promo push (delegate email sequence)
- Confirm sponsors, finalize sponsor deliverables
- Prepare presentation materials, printed programs
- Arrange catering headcount + dietary
- Assign day-of roles (registration, stage manager, AV, host, volunteers, sponsor liaison)

**1 week before:**
- Send attendee reminder with parking/arrival details
- Run tech walkthrough on-site if possible
- Finalize handoff to `event-run-of-show`
- Confirm all vendors, print attendee list + name tags
- Walk the room; mark power, WiFi, restrooms, emergency exits

**Day of:**
- Arrive 90 min early
- Final AV test, mic check, audio levels
- Brief volunteers and ushers
- Set up registration desk + swag + signage
- Test backup plan (speaker no-show, AV failure, weather)

**After:**
- Thank-you email within 24 h
- Feedback survey within 48 h (see `templates.md`)
- Compile metrics (attendance, NPS, leads, ROI)
- Repurpose content (photos, quotes, clips)
- Follow up with leads within 72 h

## High-level day-of schedule format

Each segment lists: **time • segment name • owner • duration • notes**. No minute-by-minute cues here — hand those to `event-run-of-show`.

Example:
```
09:00  Registration + coffee     Owner: Registration lead   60m
10:00  Welcome + housekeeping    Owner: MC                  10m
10:10  Keynote                   Owner: Speaker 1           40m
10:50  Break                     Owner: Catering            15m
...
```

## Sponsor / partner handling

- Sponsor tiers collected in budget tracker
- Each sponsor has a liaison assigned day-of
- Sponsor mentions scripted into MC notes (hand to `event-run-of-show`)
- Sponsor booths mapped on venue layout

## In-person contingencies (planning-level)

- Weather: indoor backup or reschedule policy announced at registration
- Speaker no-show: MC extends networking, pre-briefed backup segment
- Venue issue: coworking / partner office / pivot to virtual (Zoom link on standby)
- Low attendance: compress agenda, merge sessions, keep closing intact

Pass full contingency cues to `event-run-of-show`.
