# Google Business Profile — Field-by-Field Optimization Checklist

Work top to bottom. Do not skip to Posts before categories are right.

## 1. Verification

- [ ] Listing verified (postcard / phone / email / video / instant)
- [ ] Ownership (not just management) confirmed
- [ ] Two-factor on the owner Google account
- [ ] No duplicate listings for same address — if found, request merge or removal

## 2. Business Name

- [ ] Legal registered name ONLY — matches signage, storefront, business license
- [ ] No city, keywords, taglines, or descriptors appended ("Smith Dental — Best Lisbon Dentist" = violation)
- [ ] Special characters match real-world usage (& vs "and")
- [ ] Accents / diacritics preserved (São, Açaí, Málaga)

Suspension risk: name stuffing is the #1 cause of GBP suspension.

## 3. Categories

- [ ] Primary category is the SINGLE most accurate — this is the highest-impact ranking field
- [ ] Research competitors' primary category (use GMBspy / PlePer Chrome extension)
- [ ] Secondary categories: up to 9, but only add if genuinely offered
- [ ] Avoid broad categories when a specific one exists ("Dentist" > "Doctor"; "Pizza Restaurant" > "Restaurant")
- [ ] Review quarterly — Google adds new categories regularly

## 4. Description (750 characters)

- [ ] First 250 chars contain primary service + city (above the fold on mobile)
- [ ] No URLs, no phone numbers, no HTML, no promotional language ("best", "#1", "cheapest")
- [ ] Mention 2–3 secondary keywords naturally
- [ ] Include neighborhoods / districts served
- [ ] Close with differentiator (years in business, certifications, languages)
- [ ] Read aloud — if it sounds like ad copy, rewrite

## 5. Contact & Location

- [ ] NAP matches website footer and Schema byte-for-byte
- [ ] Phone: local number preferred over toll-free for local ranking
- [ ] Website URL uses the canonical version (https, trailing-slash consistency)
- [ ] Appointment URL (if different from main site)
- [ ] Pin is on the actual building entrance (drag to correct if off)

## 6. Service Area (SAB only)

- [ ] Address hidden ("I deliver goods and services to my customers")
- [ ] Up to 20 cities / regions listed — city names preferred over radius
- [ ] Cities are where you actually serve, not aspirational markets

## 7. Hours

- [ ] Regular hours accurate to the minute
- [ ] Special hours set for every public holiday 12 months out
- [ ] "More hours" for distinct services (delivery, kitchen, happy hour, drive-thru)
- [ ] Temporary closure flag used instead of deleting listing when closed

## 8. Services / Menu / Products

- [ ] Every service offered listed with custom service (not just predefined)
- [ ] Each service has a 300-char description with local keywords naturally placed
- [ ] Price or price range filled when possible
- [ ] Menu (restaurants): full menu with items, descriptions, prices, photos per item
- [ ] Products (retail): minimum 10 products with images, prices, descriptions

## 9. Photos

- [ ] Logo (square, 250x250+)
- [ ] Cover photo (landscape, showcases brand)
- [ ] Exterior: 3+ angles, different times of day
- [ ] Interior: 5+ covering all customer-facing areas
- [ ] Team: 2+ humanizing shots
- [ ] Products / food / work samples: 10+
- [ ] Geotagged (EXIF) — marginal benefit but free to do
- [ ] Upload cadence: 1–3 new photos per week
- [ ] No stock photos, no watermarks, no text overlays (Google may reject)

## 10. Attributes

- [ ] Every applicable attribute toggled (wheelchair accessible, wifi, outdoor seating, women-owned, LGBTQ+ friendly, appointment required, accepts cards)
- [ ] Health & safety attributes updated if relevant
- [ ] Attributes change by category — recheck after any category edit

## 11. Posts

- [ ] Weekly cadence minimum (Posts expire after 7 days for Offers/Events)
- [ ] Rotate types: Update, Offer, Event, Product
- [ ] Each post: image + 150–300 chars + CTA button
- [ ] UTM-tagged URLs for attribution in GA4

## 12. Q&A

- [ ] Seed 5–10 common questions yourself (use owner account, answer from same)
- [ ] Upvote correct answers
- [ ] Monitor weekly — flag spam/misleading answers

## 13. Reviews

- [ ] Response SLA: 48h, 100% of reviews
- [ ] See `review-workflow.md` for templates

## 14. Messaging & Booking

- [ ] Messaging enabled with <24h response time commitment
- [ ] Booking integration (Reserve with Google) where supported

## Post-Launch Monitoring

- [ ] Weekly: new reviews, Q&A, suggested edits, Post performance
- [ ] Monthly: Insights export (searches, views, actions), photo views
- [ ] Quarterly: category audit, competitor GBP snapshot, attribute review
