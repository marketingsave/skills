# Cohort Mode — Deep Reference

Fixed start/end dates, linear week-by-week curriculum, graduation milestone.

## Brief — Mode-Specific Inputs

- Graduation criteria (attendance %, assignments submitted, final project)
- Week-by-week learning objectives

## Cohort Program Brief Template

```
## Cohort Program Brief
**Program:** [Name]
**Transformation:** [From X to Y in Z weeks]
**Duration:** 6 weeks
**Cohort size:** 25 participants
**Format:** Weekly 90-min live call + async lessons + community
**Price:** $497
**Graduation criteria:** [Specific deliverable or milestone]
```

## Curriculum — Linear Weekly Rhythm

Each week follows this rhythm:

```
## Week [N]: [Theme]
**Learning objective:** [One clear outcome]
**Async content:** [Lesson topic, estimated time — cap at 60 min/week]
**Live call agenda (90 min):**
  - Hot seat (15 min)
  - Teaching segment (30 min)
  - Breakout exercise (20 min)
  - Q&A (15 min)
  - Week preview + homework (10 min)
**Homework:** [Specific deliverable]
**Peer activity:** [What participants do together]
```

Build at least three accountability systems:
1. **Accountability pods** — 3-5 members who check in weekly
2. **Homework deadlines** — specific due dates with public sharing
3. **Progress tracker** — visible dashboard
4. **Hot seats** — live coaching slots requiring preparation
5. **Peer review** — feedback on each other's work

## Operations

**Recruitment:**
- Enrollment window with application deadline
- Launch sequence to existing audience + waitlist capture for next cohort
- Cap enrollment; create urgency via close date

**Onboarding (mode-specific):** accountability pod assignments, homework submission system, 6-lesson schedule.

**Engagement Calendar Example:**
```
Week 1: Welcome email → Lesson 1 → Pod assignment → Live call → Homework 1
Week 2: Lesson 2 → Pod check-in → Live call → Homework 2 → Midweek prompt
...
```

## Assessment — Graduation Criteria

Define exactly what "completing" means:
- Minimum attendance (e.g., 4 of 6 live calls)
- Homework completion (e.g., 5 of 6 assignments submitted)
- Final project or deliverable submitted
- Optional: peer feedback given on at least 3 classmates' work

**Post-cohort:**
- Graduation celebration (live call or async thread)
- Feedback survey
- Alumni community or next-step offer
- Testimonial collection sequence

## Cohort Health Metrics

| Metric | Target |
|--------|--------|
| Attendance | 70%+ live calls |
| Completion | 80%+ homework |
| Satisfaction (NPS) | 40+ |
| Renewal / next-cohort | 20%+ alumni |

## Anti-Patterns (Cohort)

- **Content dump without interaction** — a cohort without peer elements is just a course with a deadline.
- **Too much content per week** — cap async at 60 min/week.
- **No graduation criteria** — engagement drops after week 2.
- **No async backup** — life happens. Record live calls.
- **Skipping the welcome packet** — disengagement from day one.

## Recovery (Cohort)

- **User wants 12-week cohort**: warn about fatigue. Suggest splitting into two 6-week levels or mid-program break.
- **Cohort size under 10**: skip breakouts, full-group discussion, pods of 2-3 instead of 4-5.
