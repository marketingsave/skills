# Lead Magnet Templates by Type

Every type follows the same skeleton: **Title → "Who This Is For" (one sentence) → Core Content → "What to Do Next"**. What varies is the core content block. Use the skeleton below as the fill-in template for the chosen type.

---

## 1. Checklist

**Best for:** step-by-step processes, launch sequences, routines, audits.
**Length:** 1 page. **Items:** 15-25 checkboxes grouped into 3-7 named steps.

### Skeleton

```
# [Title: The [Number]-Point [Topic] Checklist]

## Who This Is For
[One sentence naming the specific reader + specific situation.]

## Step 1: [Verb + Noun]
- [ ] [Action verb] [concrete thing to check or do]
- [ ] [Action verb] ...
- [ ] [Action verb] ...

## Step 2: [Verb + Noun]
- [ ] ...

## Step 3-7: [repeat]

## Quick Reference
| [Category] | [What to Do] | [When / Where] |
|------------|--------------|----------------|
| ...        | ...          | ...            |

## What to Do Next
[One sentence pointing to the desired action + link.]
```

### Rules

- Every checkbox item starts with an action verb (Write, Send, Check, Remove, Set up, Confirm).
- First 3 items include a quick win (under 5 minutes, immediate visible result).
- Step names are verb-first and describe an outcome, not a topic.

---

## 2. Cheat Sheet

**Best for:** reference material, rules, formulas, shortcuts.
**Length:** 1-2 pages. **Centerpiece:** scannable Quick Reference Table.

### Skeleton

```
# [Title: The [Topic] Cheat Sheet]

## Who This Is For
[One sentence.]

## Quick Reference Table
| Category | What to Do | Example |
|----------|-----------|---------|
| ...      | ...       | ...     |
(5-15 rows)

## The [N] Rules
1. **[Rule name]** — [1-2 sentence explanation + why it matters]
2. ...
(3-7 rules)

## Common Mistakes
- **[Mistake]** — [what goes wrong + the fix in one line]
- ...

## What to Do Next
[Desired action + link.]
```

### Rules

- Table is the centerpiece — it must be complete, scannable, and self-contained.
- Every row's Example column is concrete (a real line, number, or named thing).
- Common Mistakes section is high perceived value for zero extra pages — always include.

---

## 3. Mini-Guide

**Best for:** teaching a framework, explaining a method, building authority.
**Length:** 2-3 pages. **Key feature:** named framework with memorable label.

### Skeleton

```
# [Title: The [Framework Name] — A [Topic] Guide]

## Who This Is For
[One sentence.]

## Why This Matters
[2-3 sentences naming the pain, the cost of not solving it, what the framework fixes.]

## The [Framework Name] Framework

### Part 1: [Name]
[2-3 paragraphs. Explain what, why, how.]

### Part 2: [Name]
[2-3 paragraphs.]

### Part 3: [Name]
[2-3 paragraphs.]

## Real-World Example
[Walkthrough applying all 3 parts to one concrete case — named person or scenario.]

## What to Do Next
[Desired action + link.]
```

### Rules

- Framework gets a proprietary-sounding name (e.g., "The 3-Block Method", "The RISE Framework").
- Each part is actionable, not theoretical — reader can do it after reading.
- Real-World Example is not optional; it is what turns the framework from abstract to usable.

---

## 4. Template

**Best for:** fill-in-the-blank documents, scripts, email sequences, proposals.
**Length:** 1-2 pages. **Key feature:** filled-in example showing what "done" looks like.

### Skeleton

```
# [Title: The [Use Case] Template]

## Who This Is For
[One sentence.]

## How to Use This Template
- [Instruction 1]
- [Instruction 2]
- [Instruction 3-5]

## The Template
```
[BLANK TEMPLATE with labeled fields in brackets]
[Subject: ___________]
[Opening: ___________]
[...]
```

## Filled-In Example
```
[The same template, filled in for a specific scenario.]
```

## What to Do Next
[Desired action + link.]
```

### Rules

- Blank template has labeled slots (e.g., `[Prospect name]`, `[Specific pain point]`), not generic `[XYZ]`.
- Filled example uses a believable, specific scenario — not "John at Acme Corp".
- Instructions section tells reader exactly what to change for their use case.

---

## 5. Swipe File

**Best for:** proven examples to model (emails, headlines, CTAs, ad copy, hooks).
**Length:** 2-3 pages. **Always include:** "adapt to your voice, never copy verbatim" note.

### Skeleton

```
# [Title: The [Topic] Swipe File]

## Who This Is For
[One sentence.]

## How to Use This Swipe File
- Read for pattern, not for copy-paste.
- Adapt every example to your own voice and context.
- [Any topic-specific instruction]

## Category 1: [Name]
### Example 1
[The actual swipe — email, headline, hook.]
**Why it works:** [1-2 sentences on the mechanism.]

### Example 2
[...]
**Why it works:** [...]

## Category 2-5: [repeat]

## What to Do Next
[Desired action + link.]
```

### Rules

- 3-5 categories, each with 2-4 examples.
- Every example is paired with a "Why it works" note — this is the teaching layer.
- Explicit anti-copy warning in the instructions section is non-negotiable (avoids reputation damage to the user).

---

## 6. Bonus Types (advanced, less common)

Use only if the user explicitly requests one. Each has higher production cost than the 5 core types above.

| Type | Core Content | Length | When to Use |
|------|--------------|--------|-------------|
| **Quiz / Assessment** | 8-12 questions + scored result tiers + tier-specific advice | 2-3 pages | Audience segmentation, personalized follow-up |
| **Challenge (5-day / 7-day)** | One short daily action + rationale + expected outcome | 1 page/day | Building habit or momentum, email sequence native |
| **Webinar Replay / Video** | Recorded session + companion notes PDF | Video + 1-page notes | High-ticket offers, complex topics needing demonstration |
| **Resource Library** | Curated list of tools/links/articles + one-line rationale per item | 1-2 pages | Positioning as a trusted curator in a crowded niche |

For these types, follow the same skeleton (Title → Who This Is For → Core → What to Do Next) and apply the general content rules (quick win early, action-first language, one problem solved completely).
